# Исследование: Telegram Mini Apps в нише бьюти-услуг

> Каталог услуг бьюти-мастера как Telegram Mini App
> Дата: 2026-03-28

---

## 5 примеров Telegram Mini Apps

---

### 1. YCLIENTS — онлайн-запись в салоны

**Что делает:** Крупнейшая российская CRM для бьюти-бизнеса (26 000+ заведений). Telegram Mini App поверх движка записи: клиент нажимает кнопку в боте → открывается WebApp → выбирает услугу, мастера, время → подтверждает прямо в Telegram.

**Сильные стороны:**
- Проверенный 4-шаговый флоу: услуга → мастер → дата → подтверждение
- Карточка мастера: фото, имя, специализация, рейтинг, ближайшее свободное время
- Визуальный слот-пикер (сетка по дням/часам, не выпадающий список)
- Авторизация через Telegram — номер подтягивается автоматически, нет ввода данных
- Напоминания через бота за 24 часа и 2 часа до визита

**Слабые стороны:**
- Дизайн корпоративный, холодный — белый фон, синие кнопки, ощущение банка, не «бьюти»
- Нет портфолио мастера внутри флоу записи
- Нет фильтров по мастеру если их 20+
- Медленная загрузка на слабых телефонах (тяжёлый JS)
- Нет отзывов внутри Mini App — редирект на внешний сайт

**Что сделать лучше:**
- Добавить галерею работ мастера прямо в карточке (Instagram-стиль)
- Персонализировать дизайн под бренд конкретного мастера
- Skeleton-экраны вместо белого пустого экрана при загрузке
- Встроить отзывы внутрь приложения

---

### 2. NailMaster Bot — частные мастера на конструкторах

**Что делает:** Самописные или конструкторные боты (Botmaker, Salebot, BotHelp) для частных мастеров маникюра, бровистов, визажистов. Прайс + фото работ + календарь свободных дат + кнопка «Записаться». Клиент выбирает → оставляет заявку → мастер подтверждает вручную.

**Сильные стороны:**
- Всё в одном мессенджере — не нужно открывать сайт
- Мастер сам управляет расписанием
- Портфолио работ как галерея фото часто встроено
- Низкая стоимость запуска

**Слабые стороны:**
- Дизайн самодельный — кнопки разного размера, кричащие цвета, нет единого стиля
- Нет подтверждения в реальном времени — клиент ждёт ответа мастера
- Нет отмены/переноса через приложение — нужно писать мастеру отдельно
- Календарь часто — просто список дат текстом, неудобно
- Напоминания работают нестабильно

**Что сделать лучше:**
- Нормальный визуальный календарь (сетка месяца или лента дат)
- Автоматическое подтверждение + отмена/перенос одной кнопкой
- Занятые/свободные слоты цветом, а не текстом
- Единый дизайн-стиль под нишу бьюти

---

### 3. TopGun / Franky Barbershop — сетевые TMA

**Что делает:** Сети барбершопов запустили TMA для записи с программой лояльности. Бот + Mini App: запись, история визитов, баллы, уровни клиента.

**Сильные стороны:**
- Тёмная тема — бежевые/золотые акценты на чёрном фоне, выглядит дорого и атмосферно
- Карточка мастера = полноценная «визитка»: большое фото, стаж, специализация, рейтинг
- Горизонтальная карусель работ прямо на карточке (5-8 фото)
- «Занят до...» — создаёт эффект дефицита, мотивирует записаться быстрее
- Программа лояльности прямо внутри TMA

**Слабые стороны:**
- Нет поиска по стилю — хочешь конкретное, ищешь глазами по фото
- Фото нельзя открыть на весь экран и рассмотреть детали
- Запись только в «свой» барбершоп, нет агрегирования
- Нет автоматических напоминаний за 2 часа
- Нативная кнопка «Назад» Telegram часто ломает флоу у самописных TMA

**Что сделать лучше:**
- Открытие фото на весь экран с зумом
- Автоматические напоминания через Telegram Bot API
- Правильное использование `Telegram.WebApp.BackButton`

### 4. FitBase / Fitness TMA — запись на тренировки

**Что делает:** Платформы для фитнес-клубов — расписание групповых тренировок, запись, покупка абонементов, трекинг посещений внутри Telegram.

**Сильные стороны:**
- Расписание как цветной календарь — тип занятия закодирован цветом
- Счётчик мест в реальном времени: «Осталось 3 места» — мощный конверсионный триггер
- Быстрая запись одним тапом — нашёл, тапнул, подтверждено
- История посещений и прогресс клиента
- Оплата через Telegram Payments без редиректа

**Слабые стороны:**
- Перегруженный экран расписания при большом количестве занятий
- Нет фото тренера в расписании — имя есть, лица нет
- Одинаковый UX для новичка и постоянного клиента
- Приложение ломается без интернета — нет кэширования

**Что сделать лучше:**
- Адаптировать UX для нового клиента vs постоянного (персонализация)
- Фото мастера везде, где есть его имя
- Цвет-кодинг категорий услуг в каталоге бьюти

### 5. Доставка еды (Wolt TMA, Кухня на районе) — витрина + быстрый заказ

**Что делает:** TMA-версии сервисов доставки — полноценный каталог блюд, корзина, оплата, трекинг заказа внутри Telegram.

**Сильные стороны:**
- Фото занимает 60% карточки — «ешь глазами», высокая конверсия
- Горизонтальные чипы-категории в шапке — быстрое переключение без скролла
- Floating-кнопка корзины — всегда видна сумма и количество позиций
- «Повторить заказ» — один тап для повторной покупки любимого набора
- Fragment (TMA от команды Telegram) — минималистичный эталон дизайна

**Слабые стороны:**
- Перегруженный онбординг — сразу просят геолокацию, уведомления, номер
- Большие фото = медленная загрузка на слабом интернете
- Нет «истории просмотров» — вернуться к позиции которую смотрел нельзя

**Что сделать лучше:**
- Запрашивать разрешения только в момент когда они реально нужны
- WebP + lazy loading для фото
- «Недавно смотрела» как блок на главной

## Что мы можем сделать лучше всех

| Проблема у всех | Наше решение |
|---|---|
| Нет портфолио в флоу записи | Фото работ прямо в карточке мастера, открываются на полный экран |
| Медленный путь до записи | Максимум 4 тапа от входа до подтверждения |
| Нет поиска по стилю | Теги на работах мастера (объём, окрашивание, нюд и т.д.) |
| Нет отмены/переноса | Кнопка «Перенести / Отменить» в разделе «Мои записи» |
| Нет социального доказательства | Рейтинг + число работ прямо на карточке, отзывы внутри TMA |
| Дизайн одинаковый у всех | Персонализация: цвет/фото обложки под стиль конкретного мастера |
| Один UX для всех | Разный онбординг для нового и постоянного клиента |

|---|---|---|
| 1 | Splash / Welcome | Первый запуск, загрузка данных |

| 3 | Каталог услуг | Категории + карточки услуг с ценами |
| 4 | Карточка услуги | Фото, описание, цена, длительность, кнопка «Записаться» |

| 7 | Подтверждение | Итог: услуга + мастер + дата + цена |
| 8 | Экран успеха | Анимация ✓ + детали записи |

| Чат с мастером | Для доверия и вопросов перед записью |
| Программа лояльности | Когда есть постоянные клиенты |

## Обязательные UI элементы

**Навигация:**
- Bottom Tab Bar: Главная / Каталог / Мои записи / Профиль
- `Telegram.WebApp.BackButton` — нативная кнопка назад, НЕ своя
- `Telegram.WebApp.MainButton` — главное действие внизу (Записаться, Подтвердить)

**Карточка услуги:**
- Фото (минимум 200×200, формат 3:4 или квадрат)
- Название, описание (1-2 строки), цена, длительность

**Карточка мастера:**
- Аватар круглый (64px+), имя, специализация
- Рейтинг со звёздами + число работ
- Горизонтальный скролл портфолио (5-8 фото)
- Значок «Доступен сегодня»

**Слот-пикер:**
- Горизонтальная лента дат (7-14 дней)
- Сетка слотов с цветовым кодом: свободно / занято / выбрано
- НЕ полноэкранный календарь месяца

**Обратная связь:**
- Skeleton-экраны при загрузке
- `HapticFeedback` на нажатие кнопок
- Анимация на экране успеха

## Дизайн-паттерн для бьюти: «Soft Premium»

**Цвет:** Нейтральная база — беж `#F5EFE7`, крем, белый + 1 акцентный цвет (пудровый розовый, шалфей, мокко). Никаких ярких `#FF0000` или `#0000FF`.

**Типографика:** Заголовки 24-28px с засечками (Playfair, Cormorant) + body 15-16px sans-serif (Inter). Минимальный размер текста — 13px.

**Карточки:** Скруглённые углы 12-16px, мягкие тени `0 4px 16px rgba(0,0,0,0.08)`, белый фон на кремовом фоне страницы.

**Фото работ:** Вертикальный формат 3:4 — стандарт Instagram/Pinterest-эстетики, которую ожидает бьюти-аудитория.

## Типичный User Flow

```
ВХОД В MINI APP
      ↓
[Главная] — фото мастера + категории услуг + популярные
      ↓                    ↓
[Каталог услуг]     [Профиль мастера]  ← 2 точки входа
      ↓                    ↓
[Карточка услуги]   [Галерея работ + список услуг]
      ↓                    ↓
      └──────────[Выбор даты]──────────┘
                       ↓
                 [Выбор слота]
                       ↓
              [Экран подтверждения]
                       ↓
           [MainButton "Подтвердить запись"]
                       ↓
            [Экран успеха + уведомление в бот]
                       ↓
               [Мои записи — видна бронь]
```

**Правило:** от входа до кнопки «Подтвердить» — не более 4 тапов.

## Критические ошибки которых нужно избежать

1. **Не использовать `MainButton`** — делать обычную HTML-кнопку вместо нативной Telegram
2. **Игнорировать тему Telegram** — не адаптировать под `colorScheme` пользователя (light/dark)
3. **Не учитывать safe area** — контент уезжает под нотч; нужен `expand()` и `env(safe-area-inset-bottom)`
4. **Длинный путь до записи** — 5+ экранов убивают конверсию
5. **Нет skeleton-загрузчиков** — пустые экраны воспринимаются как баги
6. **Запрашивать все разрешения при старте** — геолокация, контакт, уведомления сразу = закрытие приложения
7. **Не сохранять состояние** — при сворачивании TMA состояние теряется; нужен `localStorage` или `CloudStorage`
8. **Нет пустых состояний** — раздел «Мои записи» у нового пользователя должен объяснять что делать
9. **Нет `HapticFeedback`** — кнопки без тактильного отклика ощущаются «мёртвыми»
10. **Не использовать `BackButton` Telegram** — своя кнопка «назад» ломает навигацию

## Экспертная оценка

---

### UX-дизайнер: как не облажаться с интерфейсом

**Ошибки новичков:**

- **Делают сайт внутри Telegram.** Копируют десктопный макет в мобиль — мелкие кнопки, горизонтальные таблицы, hover-эффекты. В TMA всё управляется большим пальцем, минимальная зона тапа — 44×44px.
- **Перегружают главную.** Хотят показать всё сразу: акции, категории, мастеров, новости, контакты. Пользователь не понимает что делать и закрывает.
- **Забывают про пустые состояния.** Новый пользователь открывает «Мои записи» — видит пустой белый экран. Нет иллюстрации, нет подсказки, нет CTA. Ощущение что приложение сломано.
- **Нет иерархии в типографике.** Весь текст одного размера — заголовок неотличим от описания, цена теряется среди текста.
- **Игнорируют состояния загрузки.** Пустой белый экран на 1-2 секунды = пользователь думает что завис и закрывает.

**Что обязательно должно быть:**

- **Одно главное действие на экране.** Главная — войти в каталог. Карточка мастера — записаться. Слот-пикер — выбрать время. Не распылять внимание.
- **Визуальная иерархия в карточке:** фото → название → цена (крупно) → длительность (мелко) → кнопка.
- **Состояния всех элементов:** normal / pressed / disabled / loading / empty — для каждого интерактивного элемента.
- **Touch-friendly отступы.** Между кнопками минимум 8px, сами кнопки минимум 48px высотой.
- **Плавные переходы между экранами.** Резкие появления экранов ощущаются как баг. Достаточно 200-300ms slide-анимации.

**Что бесит пользователей больше всего:**

- Приложение спрашивает номер телефона, хотя Telegram его уже знает
- Нельзя отменить запись — нужно писать мастеру в личку
- Фото работ мелкие, нельзя увеличить
- После записи непонятно что произошло — нет подтверждения, нет уведомления
- Кнопка «Назад» в Telegram закрывает всё приложение вместо возврата на предыдущий экран
- Приложение не помнит где ты был — при повторном открытии снова с главной

### Маркетолог: что заставит записаться

**Триггеры, которые реально работают:**

- **Дефицит слотов.** «Осталось 2 окна на этой неделе» — не «много» и не «мало», а конкретное число. Работает лучше чем любой баннер.
- **Социальное доказательство в моменте.** Рейтинг ★4.9 (127 работ) прямо на карточке — до того как человек провалился внутрь. Принятие решения идёт ещё при скролле каталога.
- **Фото работ как главный аргумент.** В бьюти-нише решение принимается визуально. Фото должно занимать 60% карточки. Плохое фото = нет записи, сколько бы звёзд ни было.
- **Ближайшее доступное время на карточке мастера.** «Ближайшее: завтра в 14:00» — снимает барьер «а вдруг нет мест». Видишь, что есть место — проще решиться.
- **Быстрая повторная запись.** «Записаться снова» в одно нажатие для постоянных клиентов. Не заставляй искать мастера заново — конверсия повторных обращений в разы выше.

**Конверсионные элементы обязательные:**

| Элемент | Где | Почему работает |
|---|---|---|
| Рейтинг + число отзывов | На карточке в каталоге | Доверие без клика внутрь |
| «Ближайшее время» | На карточке мастера | Снижает тревогу «а есть ли места» |
| Счётчик свободных слотов | На экране выбора даты | Триггер дефицита |
| Портфолио 3:4 крупно | Профиль мастера | Визуальное решение = запись |
| Подтверждение мгновенно | После выбора слота | Убирает неопределённость |
| Напоминание за 24ч и 2ч | Через бота | Снижает отмены на 30-40% |

**Что убивает конверсию:**

- Запрос лишних данных перед первой записью (фамилия, email, дата рождения)
- «Мастер подтвердит в течение часа» — неопределённость = отказ
- Нет цены до момента подтверждения («уточняйте у мастера»)
- Перенаправление на внешний сайт для оплаты
- Нет возможности написать вопрос мастеру до записи

**Сценарий нового пользователя (самый важный):**
Человек пришёл первый раз через ссылку или QR-код. У него нет лояльности, нет доверия. Задача первого экрана — за 3 секунды показать: «здесь красиво, здесь профессионально, здесь легко записаться». Это значит: большое фото работ, имя мастера, рейтинг, одна кнопка «Записаться».

### Telegram-разработчик: что знать до старта

**Ограничения платформы:**

- **Нет постоянного фонового процесса.** TMA закрывается когда пользователь уходит из Telegram. Состояние приложения нужно сохранять в `localStorage` или `Telegram.WebApp.CloudStorage` (синхронизируется между устройствами).
- **Viewport нестабилен.** При появлении клавиатуры `viewportHeight` меняется. Если не слушать `viewportChanged`, форма ввода уедет под клавиатуру. Всегда используй `viewportStableHeight` для критичных layout-расчётов.
- **iOS Safari внутри Telegram имеет свои баги.** `position: fixed` ведёт себя непредсказуемо при прокрутке. Вместо fixed используй sticky или отдельный DOM-слой.
- **Telegram Payments требует одобрения.** Подключение платёжного провайдера (ЮКасса, Stripe) требует заявки и занимает время. Закладывай это в план.
- **Размер бандла важен.** TMA открывается внутри WebView — на Android-бюджетниках 500kb+ JS ощущается как тормоза. Цель: первый экран за < 1.5 секунды на 3G.
- **CloudStorage ограничен.** Максимум 1024 ключа, значение до 4096 символов. Для хранения истории записей и фото — нужен свой бэкенд.

**Что работает хорошо:**

- **`MainButton`** — нативная кнопка внизу экрана, всегда видна, не перекрывается клавиатурой, поддерживает `showProgress()`. Использовать для всех главных действий.
- **`BackButton`** — интегрируется в хедер Telegram автоматически. Обязательно управлять вручную: `BackButton.show()` / `BackButton.hide()` при переходах между экранами.
- **`HapticFeedback`** — `impactOccurred('light')` на тап по карточке, `notificationOccurred('success')` на подтверждение записи. Бесплатное ощущение нативного приложения.
- **`showPopup` / `showAlert`** — нативные диалоги Telegram для подтверждений. Не делай свои modal — нативные ощущаются правильно и не требуют верстки.
- **`requestContact()`** — запрашивает номер телефона одним тапом. Но только когда реально нужен, не при старте.
- **`expand()`** — открывает TMA на весь экран. Вызывать сразу при инициализации.

**Что лучше не делать:**

- **Не делай SPA с React Router на hash-роутах.** Кнопка «Назад» в Telegram будет закрывать всё приложение. Управляй историей навигации вручную через `BackButton`.
- **Не храни JWT в localStorage без проверки.** TMA получает `initData` от Telegram — проверяй подпись на бэкенде, не доверяй только клиенту.
- **Не используй `alert()` / `confirm()`.** В некоторых версиях Telegram WebView они заблокированы. Используй `Telegram.WebApp.showPopup()`.
- **Не анимируй тяжело при скролле.** Parallax, blur-эффекты при скролле — на слабых Android убивают 60fps. Бьюти-аудитория часто на бюджетных телефонах.
- **Не открывай внешние ссылки через `window.open`.** Используй `Telegram.WebApp.openLink()` — иначе в некоторых версиях Telegram ссылка просто игнорируется.

**Технический минимум для запуска:**

```js
// Инициализация
Telegram.WebApp.expand();
Telegram.WebApp.enableClosingConfirmation(); // защита от случайного закрытия

// Тема пользователя
const isDark = Telegram.WebApp.colorScheme === 'dark';

// Главная кнопка
Telegram.WebApp.MainButton.setText('Подтвердить запись');
Telegram.WebApp.MainButton.show();
Telegram.WebApp.MainButton.onClick(handleConfirm);

// Тактильный отклик
Telegram.WebApp.HapticFeedback.impactOccurred('light'); // тап по карточке
Telegram.WebApp.HapticFeedback.notificationOccurred('success'); // успешная запись

// Safe area
document.body.style.paddingBottom = 'env(safe-area-inset-bottom)';
```

### Итоговый список рекомендаций

**Приоритет 1 — делать обязательно:**

1. Фото работ мастера 3:4 крупно — главный конверсионный элемент, не экономить на качестве
2. Максимум 4 тапа от входа до подтверждения записи
3. Автоматическое подтверждение записи (не «мастер ответит»)
4. Напоминания через бота за 24 часа и 2 часа
5. Кнопка «Отменить / Перенести» в разделе «Мои записи»
6. `MainButton` и `BackButton` — только нативные Telegram, никакой самодеятельности
7. Skeleton-экраны при любой загрузке данных
8. `HapticFeedback` на все основные действия
9. `expand()` при инициализации + safe area padding
10. Сохранять состояние в `localStorage` — не терять прогресс при сворачивании

**Приоритет 2 — сделает лучше конкурентов:**

11. «Ближайшее доступное время» прямо на карточке мастера в каталоге
12. Счётчик свободных слотов на экране выбора даты («Осталось 3 окна»)
13. Открытие фото на полный экран с зумом (pinch-to-zoom)
14. Рейтинг + число работ видны в каталоге без захода в карточку
15. «Повторить запись» в 1 тап для постоянных клиентов
16. Разный онбординг для нового и вернувшегося пользователя
17. Пустые состояния с иллюстрацией и CTA (не пустой экран)
18. Адаптация под тему Telegram (light/dark через `themeParams`)

**Приоритет 3 — после MVP:**

19. Поиск услуг по тегам (стиль, техника, длина)
20. Отзывы внутри TMA (не редирект)
21. Программа лояльности — баллы за визиты
22. Чат с мастером перед записью
23. Telegram Payments для оплаты без редиректа
