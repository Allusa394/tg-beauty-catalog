# Brief: Каталог услуг бьюти-мастера — Telegram Mini App

> Версия 1.0 (MVP)
> Дата: 2026-03-28

---

## Концепция

Telegram Mini App для частного бьюти-мастера или небольшой студии.
Клиент открывает приложение → видит услуги и работы мастера → записывается.
Всё внутри Telegram, без перехода на сайт, без регистрации.

**Главный принцип:** максимум 4 тапа от входа до подтверждения записи.

---

## Навигация (Bottom Tab Bar)

```
[ Главная ]  [ Услуги ]  [ Мои записи ]  [ О мастере ]
```

4 вкладки. Всегда видны внизу экрана. Активная вкладка выделена акцентным цветом.

**Что видит пользователь:**

## Схема переходов

```
Splash
  └→ Главная
       ├→ Каталог услуг
       │    └→ Карточка услуги
       │         └→ Выбор даты
       │              └→ Подтверждение
       │                   └→ Успех → Мои записи
       ├→ Мои записи
       │    ├→ Выбор даты (перенос / повторная запись)
       │    └→ Каталог услуг (пустое состояние)
       ├→ О мастере
       │    ├→ Просмотр фото (полный экран)
       │    ├→ Отзывы
       │    └→ Каталог услуг (по «Записаться»)
       └→ Просмотр фото (с главной / карточки услуги)
```

## Чего НЕ БУДЕТ в первой версии

| Что | Почему отложено |
|---|---|
| Поиск по услугам | Мало услуг в MVP, не нужен |
| Фильтры мастеров | Один мастер в MVP |
| Чат с мастером | Сложная интеграция, есть Telegram-чат рядом |
| Программа лояльности | После первых постоянных клиентов |
| Telegram Payments (оплата) | Требует одобрения, добавить в v2 |
| Тёмная тема | После основного дизайна |
| Выбор мастера | Один мастер, не нужен экран выбора |
| Регистрация / профиль пользователя | Авторизация через Telegram автоматически |
| Push-уведомления с настройкой | Напоминания через бот без настроек |
| Агрегатор нескольких мастеров | Только один мастер |
| Онбординг-слайды | Не нужны для простого приложения |
| Сторис | После MVP |
| Рекомендации «вам может понравиться» | После MVP |

## Технические требования

**Платформа:** Telegram Mini App (WebApp API)

**Обязательные API:**
```
Telegram.WebApp.expand()                    // открыть на весь экран
Telegram.WebApp.MainButton                  // главная кнопка
Telegram.WebApp.BackButton                  // кнопка назад
Telegram.WebApp.HapticFeedback              // тактильный отклик
Telegram.WebApp.showPopup()                 // диалоги подтверждения
Telegram.WebApp.openLink()                  // внешние ссылки
Telegram.WebApp.CloudStorage               // сохранение состояния
```

**Дизайн-токены:**
```
Фон страницы:     #F5EFE7  (беж)
Фон карточки:     #FFFFFF  (белый)
Акцентный цвет:   #C8A882  (мокко / пудровый)
Текст основной:   #1A1A1A
Текст вторичный:  #8A8A8A
Ошибка/занято:    #E0E0E0
Успех:            #4CAF50
Радиус карточки:  16px
Тень карточки:    0 4px 16px rgba(0,0,0,0.08)
Высота кнопок:    48-56px (минимум 44px)
```

**Шрифты:**
- Заголовки (H1): 22-24px, 700
- Подзаголовки (H2): 17-18px, 600
- Текст (body): 15-16px, 400
- Мелкий текст: 13px, 400 (минимум!)

**Хранение данных:**
- `localStorage` — текущий выбор (услуга, дата, слот)
- `CloudStorage` — история записей пользователя
- Бэкенд — расписание мастера, список услуг, фото, отзывы

**Язык интерфейса:** Русский (все тексты, кнопки, подписи)

## Что делает это приложением, а не сайтом

- `expand()` при старте — занимает весь экран телефона
- `MainButton` вместо HTML-кнопки — нативно вписана в интерфейс Telegram
- `BackButton` в хедере Telegram — не своя стрелка в верстке
- `HapticFeedback` — приложение «отзывается» на прикосновения
- `showPopup()` вместо `alert()` — нативные диалоги
- Нет горизонтального скролла страницы
- Нет zoom на двойной тап (meta viewport)
- Нет синей рамки выделения на тапах (`-webkit-tap-highlight-color: transparent`)
- Анимации переходов между экранами (не резкое появление)
- Touch-цели минимум 44×44px везде
