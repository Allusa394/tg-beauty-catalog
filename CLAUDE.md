# CLAUDE.md — Документация проекта

## Что это

Telegram Mini App — каталог услуг бьюти-мастера с онлайн-записью.
Технологии: HTML + CSS + JavaScript, без фреймворков.

---

## Структура файлов

```
tg-beauty-catalog/
├── tg-app/               ← папка с приложением
│   ├── index.html        ← точка входа, HTML-структура всех экранов
│   ├── style.css         ← все стили, темы, анимации

├── brief.md              ← техническое задание: экраны, переходы, требования
└── CLAUDE.md             ← этот файл
```

## Файлы — за что отвечает каждый

### `tg-app/index.html`
Только структура: пустые div-обёртки для экранов и таб-бар.
Содержимое экранов рендерится через JavaScript.
Здесь НЕ надо ничего менять для изменения текста или дизайна.

### `tg-app/data.js`
**Главный файл для редактирования контента.**
- `MASTER` — имя, emoji, рейтинг, адрес, текст «О себе»
- `SERVICES` — массив услуг с ценами, описаниями, emoji
- `CATEGORIES` — категории для фильтра в каталоге
- `REVIEWS` — отзывы клиентов
- `SCHEDULE` — рабочие часы и занятые слоты
- `POPULAR_SERVICE_IDS` — какие услуги показывать на главной

### `tg-app/style.css`
Все стили. Структура через комментарии-секции:
- CSS-переменные в `:root` — цвета, радиусы, тени
- `body.dark { }` — тёмная тема (применяется автоматически)
- Компоненты: карточки, слоты, чипы, кнопки

### `tg-app/app.js`
Логика приложения. Структура:
- Инициализация Telegram WebApp SDK
- Состояние `state` — текущий таб, выбранная услуга, слот, дата
- Навигация: `switchTab()`, `pushScreen()`, `goBack()`
- Рендер экранов: `renderHome()`, `renderCatalog()`, `renderBookings()`, `renderAbout()`
- Рендер push-экранов: `openService()`, `openDatetime()`, `openConfirm()`
- Действия: `confirmBooking()`, `cancelBooking()`, `rescheduleBooking()`

## Экраны и переходы

```
Splash (автоматически через 1.2 сек)
  └→ Главная [таб]
       ├→ Каталог [таб] → Карточка услуги [push]
       │                       └→ Выбор даты/времени [push]
       │                            └→ Подтверждение [push]
       │                                 └→ Успех [push] → Мои записи [таб]
       ├→ Мои записи [таб]
       │    ├→ Выбор даты (перенос)
       │    └→ Каталог (пустое состояние)
       └→ О мастере [таб]
            ├→ Отзывы [push]
            └→ Каталог (кнопка «Записаться»)
```

**Таб-экраны** — переключаются через `switchTab(tabId)`, fade-анимация.
**Push-экраны** — открываются поверх через `pushScreen(screenId)`, slide-анимация.
**Назад** — `goBack()` или нативная BackButton Telegram.

## Как изменить данные

### Поменять имя мастера
В `data.js`, объект `MASTER`:
```js
name: 'Имя Фамилия',
emoji: '👩‍🎨',        // эмодзи вместо фото
title: 'Специализация',
```

### Добавить услугу
В `data.js`, массив `SERVICES`, добавить объект:
```js
{
  id: 9,               // уникальный номер
  category: 'mani',   // id категории из CATEGORIES
  emoji: '💅',
  name: 'Название услуги',
  shortDesc: 'Короткое описание (1 строка)',
  desc: 'Полное описание для карточки',
  price: 2500,         // в рублях (число)
  duration: '1 час',
  rating: 4.9,
  works: ['💅', '✨', '🌸', '💗', '🩷', '💜'],  // emoji примеров работ
}
```

### Добавить отзыв
В `data.js`, массив `REVIEWS`:
```js
{
  id: 6,
  name: 'Имя Ф.',
  avatar: '👩',        // emoji аватара
  stars: 5,           // от 1 до 5
  text: 'Текст отзыва...',
  date: '1 неделю назад',
  service: 'Маникюр гель-лак',
}
```

### Изменить расписание
В `data.js`, объект `SCHEDULE`:
```js
workHours: ['10:00', '11:00', ...],  // рабочие часы
busySlots: {
  0: ['10:00', '14:00'],   // 0 = смещение от сегодня
  1: ['11:00'],
  ...
},
daysOff: [0],  // 0 = воскресенье, 6 = суббота
```

### Изменить акцентный цвет
В `style.css`, `:root`:
```css
--accent: #2AABEE;  /* Telegram синий по умолчанию */
```

## Как запустить локально

Открыть `tg-app/index.html` в браузере.
Без Telegram SDK некоторые функции (HapticFeedback, BackButton, showPopup) не работают, но приложение полностью функционально.

## Как подключить к Telegram

1. Создать бота через [@BotFather](https://t.me/BotFather)
2. Разместить файлы `tg-app/` на хостинге с HTTPS (GitHub Pages, Vercel, Netlify)
3. В BotFather: `/newapp` → указать URL → получить ссылку вида `t.me/bot?startapp`
4. Для локального тестирования: `ngrok http 5500` (если используешь Live Server в VS Code)

## Что делает приложение — кратко

1. **Главная** — знакомство с мастером, популярные услуги, портфолио
2. **Услуги** — каталог с фильтром по категориям, переход в карточку
3. **Карточка услуги** — описание, цена, примеры работ, кнопка записи
4. **Выбор времени** — лента дат, сетка слотов, счётчик свободных окон
5. **Подтверждение** — итоговая карточка записи с адресом
6. **Успех** — анимация, детали, кнопка в «Мои записи»
7. **Мои записи** — активная запись с кнопками отмены/переноса, история
8. **О мастере** — портфолио-сетка, отзывы, ссылка на все отзывы
